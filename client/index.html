# Tymio — note for automated fetchers (SPA shell)

This response is a JavaScript single-page app. Without executing the bundled script, you will only see an empty #root and the document title. That is normal.

## Product
Multi-tenant product management: workspaces, initiatives, features, requirements, delivery boards, accounts, partners, demands, campaigns.

## Humans
Use a browser with JavaScript. Workspace sign-in typically uses paths like /t/<workspace-slug> on this host.

## Agents / integrations
- MCP (Streamable HTTP): POST/GET /mcp on this origin; OAuth or Bearer per deployment; workspace via session or X-Tenant-Id when supported.
- GET /api/mcp/agent-context — public JSON with short agent guidance (e.g. feedback reporting).
- REST: /api/* — requires authentication (not a public scrape API).
- Ontology + coding-agent guide (auth): /api/ontology, /api/agent/coding-guide (Markdown) — after login/token per deployment.

## Documentation discipline
Maintain a doc in THIS codebase/repo (e.g. docs/tymio.md or TYMIO.md): base URLs, workspace slug/tenant id, MCP config, key API paths, ontology/guide endpoints, and pointers to product requirements inside Tymio (initiative titles, IDs, or links). Update when integration changes. Do not rely only on chat context — the documentation repository is the durable record.
